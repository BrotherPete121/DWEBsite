Hosting Your Dweb3 Domain @ Github

I recently mapped out the process for building awesome Dwebsites and hosting them on Github for the Web3 Domains that are registered on the Decentraweb Project’s Awesome ENS Platform. 

I was working on a related Dweb Project earlier when I discovered that there is a much easier way for us Dwebbers to create a super fly and fab’lous Dwebsite and not get our skulls cracked every time we need to update our Dweb content by using Github for hosting.

Y'all Ready for the Good Stuff?

Alright then, let’s get this deal started! First off, you’ll want to go to your DNS Admin Dashboard @ Decentraweb so you can update your DNS Settings. It is there that you’ll want to create 4 New A Records for the Web3 Domain Name that you are going to use Github to build and host on.

It is these A Records that will act as quasi-nameservers in place of actual nameservers to allow your Dwebsite to be built and hosted @ Github.

I recommend creating all 4 A Records before committing them and signing the transaction to save on Gas Fees. This is the process that I used when I mapped out building and hosting my superfly http://bitcoin.stx Web3 domain name. 

If you haven’t already customized your Chrome to view Web3 Domains, just enter chrome://settings/security into your address bar, then scroll down to “Use Secure DNS” then select custom and use Decentraweb’s Resolver Link;

https://resolver.decentraweb.org/dns-query 

This will allow you to click on and view Web3 Domains such as, well for example, (shameless plug) http://bitcoin.stx!

Getting Going On Github

After you have your Github Account created, or if you already had one and you are signed in, the next step is creating a Repository to host your Web3 Domain.

You can either do that from scratch or just import the Repository that I already have set up for this, which is likely the easier way.

This Repository is where you’ll have the needed files to bring your Dwebsite online. 

Once you are logged in to Github and on your profile page, click on Repositories on the upper left side, and it’ll bring you to a page similar to mine. Then on the right hand side of the page, click on the Green New Button.

1. How Create Your Repository

2. Click On Import

On this page you'll click on Import a Repository and it will automatically forward you to the page in Step 3.

3. Import Dweb3 Repository

To Import my Repository, just add https://github.com/web3enswizard/dweb3.git, name your Repository, select Public, and click Begin Import.

Awesome Sauce! With your Repository imported and available, you are ready to build out your website to publish!

The placeholder data in the CNAME and index.html files is set up with placeholder data for you to customize as you wish.

You’ll want to open up the CNAME File and replace the link with your own and in the index.html file the basic HTML code is built for you to customize for a simple redirect. 

Enjoying This Tutorial So Far?

Okay, now this next section is on learning how to publish your Dwebsite on Github after you’ve made your edits. 

Publishing Your Web3 Domain On Github

On your Repository Page, you’ll see a Settings in the top middle bar and go ahead and click on that and that’ll take you to the General Information page about the Repository.

Once you are there, and on the left side toward the bottom there is a Pages Link and you’ll want to go ahead and click on that.

This takes you to the page where the really cool magic pops off for your Dwebsite.

Don’t Worry! You are just about ready to deploy your Dwebsite!

Once you arrive to this page, in the Branch Section in the middle of the page, click on None and then select Main and then the Root File will pop up and just click the Save button next to it.

The page will reload after you click the Save Button and just scroll down to Custom Domain and add your Web3 Domain Address but don’t use the http or https prefix and then click on Save next to it.

TA DAA! You just published your Web3 Domain using Github to host it, ya crazy Dwebhead, so Congratulations!
